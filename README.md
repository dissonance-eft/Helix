# Helix

**Helix is a research framework for discovering structural invariants across complex systems by organizing disparate data into a unified cognitive coordinate system.**

Every input to Helix—whether it is a soundchip register stream, a mathematical conjecture, or a linguistic phrasal structure—is treated as a **projection of an underlying invariant structure**. Helix is the process of recovering these invariants under conditions of **Partial Observability**.

## Mental Model
> **"The representation is the window. The invariant is the target."**

Helix distinguishes between:
1. **The Representation**: The observable dialect (e.g., VGM, MIDI, MP3).
2. **The Invariant**: The structural truth that survives across all projections.

## Functional Pipeline
Helix operates as a single canonical flow:
**Library → Helix → Atlas**

- **Library (Storage)**: Read-only repository of reference models and priors (Sound chips, grammars, axioms).
- **Helix (Processing)**: The execution engine that decomposes inputs, extracts measurable structure, and projects them into the Cognitive Coordinate System.
- **Atlas (Memory)**: The persistent research graph that stores instantiated entities and their structural relationships.

## What Helix is NOT
To maintain structural focus, it is explicitly defined that Helix is **NOT**:
- **A Recommender System**: Optimization is focused on structural truth, not user engagement.
- **A Metadata Database**: Standard metadata is secondary; structural invariants are the primary focus.
- **A Storage System**: Helix is an analysis engine. It refers to data stored in external repositories.
- **A Subjective Analysis Tool**: Every observation must be evidence-backed and grounded in measurable signals.
